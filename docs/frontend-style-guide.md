# 前端样式约束

## 1. 总体方向

本项目的前端样式统一采用 **Ant Design 的平面化、分区式表达**，强调：

- 清晰的边框分区
- 稳定的块面层级
- 可读性优先的间距与排版
- 门户与后台一致的视觉语言

本项目 **不允许使用悬浮感样式**。

## 2. 禁止项

以下样式在整个项目中禁止使用：

- 阴影营造的悬浮卡片，如 `shadow-*`、自定义 `box-shadow`
- 毛玻璃、磨砂、玻璃态，如 `backdrop-blur`、半透明浮层叠加
- hover 抬升、位移、漂浮动画，如 `translateY`、`translate-y-*`
- 通过大圆角、厚阴影制造“漂在页面上”的独立面板
- 无边界的悬浮按钮、悬浮工具条、悬浮信息块

## 3. 推荐项

优先使用 Ant Design 自带的平面组件表达信息层级：

- `Card`
  - 使用有边框、无阴影的内容分区
- `Tabs`
  - 用于服务分类、信息切换，不额外叠加卡片阴影
- `Steps`
  - 用于流程表达，替代花哨的时间轴或漂浮模块
- `Tag`
  - 用于状态、类型、通知标签
- `Statistic`
  - 用于关键数字展示，但外层仍保持平面边框容器
- `Descriptions`
  - 用于结构化信息，不额外加玻璃态或阴影面板

## 4. 实现规则

- 所有卡片、抽屉、弹层默认应为 `shadow-none`
- 需要区分层级时，优先使用：
  - 边框
  - 背景浅色差
  - 标题分隔线
  - 留白
- 页面主容器优先使用浅底色 + 白底面板，不做漂浮叠加
- 如果需要强调重点区域，优先使用品牌色边框、左侧色条、标题字重，不使用阴影
- 新增页面或组件时，先检查是否符合本文件，再提交

## 5. 适用范围

本规则适用于：

- 首页与门户页
- 登录、注册、投稿等表单页
- 作者中心、编辑工作台、管理后台
- 所有复用 UI 组件
