# 中外法学智能编审系统 v0.1 — 初版产品需求说明（PRD）

## 一、产品定位与目标

### 1.1 产品名称与版本
- **名称**：中外法学智能编审系统（Law Journal Intelligent Editorial System）
- **初版版本**：v0.1（MVP）

### 1.2 一句话定位
面向《中外法学》等法学期刊编辑部的**投稿管理 + AI 辅助初审**平台，先解决「稿件规范收入、自动基础审核、提前筛出不规范稿件、让编辑更快判断是否进入下一步」。

### 1.3 三个关键词
| 关键词 | 含义 |
|--------|------|
| **投稿管理** | 正经业务系统：用户、权限、稿件、版本、状态流转，而非单点上传工具 |
| **AI 辅助编审** | 辅助而非替代：做格式/引注/结构/相似/AI 风险检查，把问题整理给编辑，省时间、不乱下判断 |
| **知识库支撑** | 投稿自动入库、向量化，为相似检索、相关推荐、后续智能体审稿打基础 |

### 1.4 初版成功标准
- **闭环跑通**：作者投稿 → 系统自动解析 → 生成 AI 初审报告 → 编辑查看并决定退修/继续/退稿。
- **编辑部可感知价值**：① 不再靠邮箱和文件夹手工整理投稿；② 每篇新稿自动有基础审核结果；③ 明显格式与规范问题被提前发现；④ 编辑对稿件是否值得进一步处理判断更快。

## 二、角色与权限

| 角色 | 说明 | 初版权限范围 |
|------|------|--------------|
| **编辑部管理员** | 系统配置、用户与栏目管理 | 管理用户、栏目、规则模板、基础配置、查看全局状态 |
| **责任编辑** | 处理稿件、查看报告、退修/退稿 | 稿件列表、详情、AI 初审报告、状态修改、退修意见、版本比较与筛选 |
| **作者/投稿者** | 投稿与查看本人稿件 | 注册登录、投稿表单、上传 Word/PDF、填写元信息、查看投稿状态、退修意见与修订稿上传 |
| **读者** | 初版非重点 | 仅保留：期刊介绍、投稿须知、往期目录等轻量内容 |

## 三、核心业务流程

### 3.1 主链路（MVP 闭环）
```
作者投稿（表单+文件） → 系统保存并触发解析 → 文档解析+预处理 → 知识库入库+向量化
    → 生成 AI 初审报告（规则检查+相似度+AI 风险+建议） → 编辑在工作台查看
    → 编辑操作：退修 / 继续处理 / 退稿 → 若退修：作者查看意见并上传修订稿 → 新版本入稿、可再次生成报告
```

### 3.2 稿件状态（初版）
建议状态枚举：`draft`（草稿）、`submitted`（已投稿）、`parsing`（解析中）、`under_review`（初审中）、`revision_requested`（待退修）、`revised_submitted`（已提交修订稿）、`accepted`（录用）、`rejected`（退稿）。  
（具体命名可在实现时与编辑部用语对齐。）

### 3.3 版本与留痕
- 每次作者提交（首次投稿或修订稿）生成一个**稿件版本**，保留文件与解析结果。
- 编辑的**状态变更、退修意见、操作时间**均需记录，便于审计与回溯。

## 四、功能边界：初版做与不做

### 4.1 初版必须包含

**作者端**
- 注册、登录
- 投稿表单：标题、摘要、关键词、作者信息、单位、基金项目、联系方式
- 上传稿件（Word/PDF）及可选补充材料
- 投稿声明（原创承诺、未一稿多投等）
- 查看本人投稿列表、当前状态、历史版本
- 查看退修意见、上传修订稿

**编辑端**
- 稿件列表（支持新稿、待处理、高风险、待退修等维度与筛选）
- 稿件详情页（驾驶舱）：基本信息、原文预览、摘要与关键词、引注问题列表、近似段落提示、AI 风险分析、修改建议、操作记录、编辑操作按钮
- 状态修改、退修意见记录、版本比较

**系统自动能力**
- 文档解析与预处理：提取标题、摘要、关键词、正文、脚注、参考文献、章节结构
- 引注基础检查（规则引擎 + 问题列表）
- 站内相似度检测（向量检索 + 相似段落标注）
- AI 风险提示（模板化/结构异常等，不给出简单「AI 率」）
- 修改建议生成（结构性、规范性、引注、摘要、术语等）

**管理员端**
- 用户管理、栏目管理、规则/退修意见模板、基础配置、全局状态查看（初版可轻量，但需预留）

### 4.2 初版不做（后续迭代）
- 复杂双盲外审、专家库与审稿人分配
- 全文出版与正式发行系统
- 复杂读者检索与高级门户
- 知识图谱可视化大屏
- 与商业查重平台的深度对接（先做站内近似）
- 精准「AI 率」数值、自动学术价值评分

## 六、AI 初审报告结构（输出格式约定）

以下为「报告长什么样」的约定，便于前后端与 AI 模块对齐；具体字段可在实现时用 JSON 固化。

1. **稿件概览**：标题、作者、字数、章节结构摘要。
2. **结构评估**：章节是否完整、摘要/关键词是否缺失等。
3. **引注问题**：脚注格式、法条/案例书写、中外文文献信息完整性等，逐条列出。
4. **相似风险**：站内相似段落列表（段落摘要 + 相似来源 + 相似度或等级）。
5. **AI 风险提示**：如语言模板化、论证支撑不足、引注密度异常、概念堆砌、段落结构过于均匀等，以「提示项 + 简要说明」呈现，不做单一「AI 率」结论。
6. **修改建议**：分类型（结构性、规范性、引注补充、摘要、术语表述等），每条可含建议原文或位置引用。

## 七、知识库初版范围

- **内容分类**：已发表文章、历史投稿、规则规范材料、法学基础参考材料（如法条、司法解释、典型案例摘要等）。
- **入库时机**：作者投稿后自动保存原文件、提取文本、切块、打元数据（稿件编号、版本、段落位置、标题层级、作者、投稿时间、学科方向等）、向量化并写入向量库。
- **使用方式**：支持近似度检测、相关文献扩展、后续智能体审稿调用；初版不强调知识库的可视化，以「可入库、可查、可被 AI 模块调用」为目标。

## 八、技术选型（约定）

**初版锁定组合**：Next.js + TypeScript + Ant Design（前端）/ FastAPI（后端）/ PostgreSQL + pgvector / MinIO 或 OSS / JWT 或 Session 认证；任务队列初版轻量，后续再加异步。详细理由见 `docs/tech-stack.md`。

| 层级 | 选型 | 说明 |
|------|------|------|
| 前端 | Next.js + TypeScript + Ant Design | 投稿门户与编辑后台统一；Ant Design 优先（表格、表单、审核面板多），可后续换 shadcn/ui |
| 后端 | Python FastAPI | 单体后端，模块清晰；文档解析、向量化、模型调用贴近 Python 生态 |
| 数据库 | PostgreSQL | 主库；pgvector 做向量存储，初版不另起向量库 |
| 文件存储 | MinIO / OSS（或初版本地目录） | 稿件与补充材料；后续可换 S3 |
| 文档解析 | python-docx（Word）、PyMuPDF（PDF） | 初版以规范 Word/文字型 PDF 为主 |
| AI | Embedding（向量化与相似检索）+ 大模型（报告生成、建议与解释） | 任务拆开，避免全部堆给大模型 |
| 认证 | JWT 或 Session | 初版优先简单稳定 |

**工程原则**：解析、入库、向量化、报告生成等采用**后台任务**，不阻塞上传接口；先业务后智能，先闭环后增强。

## 九、初版开发顺序建议

1. **产品原型与数据结构**：确定页面、角色、稿件状态、各页字段与流程。
2. **后端基础骨架**：用户认证、稿件上传与列表/详情、状态变更、文件存储与数据库表。
3. **文档解析链路**：从上传到输出结构化正文、摘要、脚注、章节等。
4. **知识库入库与相似度检索**：切块、向量化、存储与检索接口。
5. **AI 初审报告生成**：按上述六块结构稳定输出报告（可先规则+模板，再接模型）。
6. **编辑工作台与稿件详情页打通**：编辑可查看报告并完成状态与退修操作。
7. **管理员配置与增强**：规则模板、图谱预留、外部接口等。
